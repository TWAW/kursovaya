# IT Audit Automation

**Система автоматизации IT-аудитов оборудования для ООО «IT Лидер» (г. Липецк)**

---

## 📋 Описание проекта

IT Audit Automation — это веб-приложение для внутреннего использования в компании ООО «IT Лидер», предназначенное для автоматизации процесса проведения IT-аудитов оборудования. Система позволяет вести учет оборудования, проводить аудиты, хранить результаты проверок и формировать автоматические отчеты.

### Основные возможности (core-сценарий):

- ✅ **Учёт оборудования** — ведение реестра компьютеров, серверов, принтеров и сетевых устройств
- ✅ **Управление сотрудниками** — CRUD для ответственных лиц, привязка сотрудников к оборудованию и замечаниям
- ✅ **Проведение аудитов** — создание записей об IT-аудитах с датой, статусом и комментариями
- ✅ **Замечания по результатам аудита** — фиксация проблем с критичностью, сроками и ответственными
- ✅ **Контроль исполнения** — фильтры по статусу и критичности, отдельный режим «мои замечания»
- ✅ **Автоматические отчеты** — сводная статистика по статусам и устройствам, блок по замечаниям
- ✅ **Экспорт в Excel** — выгрузка отчёта для руководства в формат .xlsx

### Расширенные возможности (для развития и production-городности):

- 🔒 **Безопасность** — авторизация с ролями, хэширование паролей, CSRF, rate limiting, 2FA (опционально)
- 📜 **Журнал аудита действий** — логирование операций пользователей (просмотр только для администраторов)
- 🧩 **Планы аудитов** — задел для автоматизации графиков проверок и напоминаний

## 🚀 Инструкция по запуску

### Требования
- Python 3.8+
- PostgreSQL (создайте БД и пользователя, как в `app/config.py`, или задайте `DATABASE_URL`)

### Установка и запуск

```bash
python -m pip install -r requirements.txt
```

**Переменные окружения (по желанию):**
- `FLASK_ENV` — `development` (по умолчанию), `production`, `testing`
- `DATABASE_URL` — URI PostgreSQL (в production обязателен)
- `SECRET_KEY` — секрет приложения (в production обязателен)
- `ADMIN_PASSWORD` — пароль для создаваемого при первом запуске пользователя `admin` (по умолчанию `Admin@123`)

**Применить миграции:**
```bash
alembic upgrade head
```

**Запуск тестов:**
```bash
pytest
# или с подробным выводом
pytest -v
```

**Запуск:**
```bash
# Разработка
flask run
# или
python wsgi.py

# Production
gunicorn wsgi:app
```

Приложение доступно по адресу **http://localhost:5000** (порт можно задать через `PORT`).

### Первый вход

При первом запуске создаётся пользователь **admin** (если в БД ещё нет пользователей). Пароль — значение `ADMIN_PASSWORD` или `Admin@123`. На production обязательно смените пароль после первого входа.

## 💻 Пример работы

### Главная страница (/)
Отображает общую статистику:
- Количество единиц оборудования
- Количество сотрудников
- Количество проведенных аудитов
- Список последних 5 аудитов

### Страница "Оборудование" (/devices)
- Просмотр всего оборудования в виде таблицы
- Добавление нового оборудования через модальное окно
- Редактирование существующего оборудования
- Удаление оборудования (с подтверждением)
- Назначение ответственного сотрудника

### Страница "Сотрудники" (/employees)
- Просмотр всех сотрудников с количеством закреплённого оборудования
- Добавление, редактирование, удаление сотрудников (только для администраторов)
- При удалении сотрудника у оборудования автоматически снимается привязка

### Страница "Аудиты" (/audits)
- Просмотр всех проведенных аудитов
- Фильтрация по статусу (исправен/неисправен/требует внимания)
- Добавление нового аудита
- Редактирование существующих записей
- Удаление записей об аудитах

### Страница "Отчеты" (/report)
- Статистика по статусам (исправно/неисправно/требует внимания)
- Сводная таблица по каждому устройству
- Полный отчет по всем аудитам с детальной информацией
- Экспорт отчёта в Excel (.xlsx)
- Возможность печати отчета

### Страница "Журнал аудита" (/report/audit-log)
- Просмотр всех действий пользователей (только для администраторов)
- Фильтрация по действиям, пользователям, статусам
- Отображение IP-адресов, User-Agent, деталей операций

## 🔐 Безопасность

- **Пароли**: bcrypt (хеширование)
- **2FA**: TOTP (Google Authenticator), опционально по пользователю
- **Сессия**: Flask-Login, защищённые cookie
- **CSRF**: Flask-WTF
- **Rate limiting**: Flask-Limiter (в т.ч. на страницу входа), блокировка по счётчику попыток в БД
- **Аудит**: логирование входов, 2FA, смены пароля и CRUD по устройствам/аудитам в таблице `audit_logs`
- **Заголовки**: X-Frame-Options, X-Content-Type-Options, CSP

Для production: задайте `SECRET_KEY` и `DATABASE_URL` в окружении, используйте HTTPS.

### Восстановление доступа при потере аутентификатора

**Вариант 1: через веб-интерфейс**
1. Перейдите на `/recover-2fa`
2. Введите имя пользователя и пароль
3. После проверки 2FA будет отключена

**Вариант 2: через скрипт (для администраторов)**
```bash
python scripts/disable_2fa_for_user.py <username>
# Пример: python scripts/disable_2fa_for_user.py admin
```
